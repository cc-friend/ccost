# cctokens sl — Statusline Analysis Design

## Overview

New `cctokens sl` subcommand that analyzes `~/.claude/statusline.jsonl` — a file generated by a Claude Code statusline hook (`statusline.sh`) that captures per-action snapshots including rate limits, self-reported cost, duration, and context window usage.

This data source provides information unavailable in conversation JSONL files, most notably **rate limit tracking** (5-hour and 7-day quota percentages and reset times), which is critical for investigating usage quota concerns.

## Data Source

Each line in `statusline.jsonl` is a JSON object with `ts` (unix timestamp) and `data` containing:

| Field | Type | Notes |
|-------|------|-------|
| `data.session_id` | string | UUID |
| `data.workspace.project_dir` | string | Full path |
| `data.model.id` | string | e.g. `claude-opus-4-6[1m]` |
| `data.model.display_name` | string | e.g. `Opus 4.6 (1M context)` |
| `data.version` | string | Claude Code version |
| `data.cost.total_cost_usd` | f64 | Cumulative per-session, **resets on resume** |
| `data.cost.total_duration_ms` | u64 | Cumulative, resets on resume |
| `data.cost.total_api_duration_ms` | u64 | Cumulative, resets on resume |
| `data.cost.total_lines_added` | u64 | Cumulative, resets on resume |
| `data.cost.total_lines_removed` | u64 | Cumulative, resets on resume |
| `data.context_window.used_percentage` | Option<u8> | null early in session |
| `data.context_window.context_window_size` | u64 | e.g. 1000000 |
| `data.rate_limits.five_hour.used_percentage` | Option<u8> | Account-level, does NOT reset with session |
| `data.rate_limits.five_hour.resets_at` | Option<i64> | Unix timestamp |
| `data.rate_limits.seven_day.used_percentage` | Option<u8> | Account-level |
| `data.rate_limits.seven_day.resets_at` | Option<i64> | Unix timestamp |

### Session Counter Reset Behavior

When a session is closed and resumed, the cumulative counters (`cost`, `duration`, `tokens`, `lines`) reset to 0. Rate limits do NOT reset — they are account-level snapshots.

**Segment detection**: within the same `session_id`, if `cost_usd` drops below previous value (by > 0.0001) OR `duration_ms` drops below previous value (by > 100), a new segment starts.

**Correct aggregation**: per segment, take max of each cumulative field; sum across segments for session total.

**Edge case**: consecutive records where both cost and duration are 0 (session just opened, no API calls yet) are NOT treated as resets.

## Data Types

```rust
/// Parsed statusline snapshot
struct SlRecord {
    ts: DateTime<Utc>,
    session_id: String,
    project: String,
    model_id: String,
    model_name: String,
    version: String,
    cost_usd: f64,
    duration_ms: u64,
    api_duration_ms: u64,
    lines_added: u64,
    lines_removed: u64,
    context_pct: Option<u8>,
    context_size: u64,
    five_hour_pct: Option<u8>,
    five_hour_resets_at: Option<DateTime<Utc>>,
    seven_day_pct: Option<u8>,
    seven_day_resets_at: Option<DateTime<Utc>>,
}

/// Session summary after segment-aware aggregation
struct SlSessionSummary {
    session_id: String,
    project: String,
    model_name: String,
    version: String,
    segments: u32,
    total_cost: f64,
    total_duration_ms: u64,
    total_api_duration_ms: u64,
    total_lines_added: u64,
    total_lines_removed: u64,
    max_context_pct: Option<u8>,
    first_ts: DateTime<Utc>,
    last_ts: DateTime<Utc>,
    last_five_hour_pct: Option<u8>,
    last_seven_day_pct: Option<u8>,
}

/// Rate limit timeline entry (deduped to show only changes)
struct SlRateLimitEntry {
    ts: DateTime<Utc>,
    session_id: String,
    five_hour_pct: u8,
    five_hour_resets_at: DateTime<Utc>,
    seven_day_pct: u8,
    seven_day_resets_at: DateTime<Utc>,
}

/// 5-hour window summary
struct SlWindowSummary {
    window_start: DateTime<Utc>,
    window_end: DateTime<Utc>,   // = resets_at
    peak_five_hour_pct: u8,
    sessions: u32,
    total_cost: f64,
    est_budget: Option<f64>,     // total_cost * 100 / peak_pct
}
```

## Processing Pipeline

```
load_sl_records → filter → aggregate by view mode → format
```

1. **load_sl_records(file_path, options)**: parse JSONL line by line, skip malformed lines, apply time/session/project/model filters
2. **Aggregate by view mode**: see View Modes below
3. **Format**: sl-specific table/json/csv/markdown/html formatters

## View Modes

| Usage | View | Aggregation |
|-------|------|-------------|
| `cctokens sl` | Rate limit timeline | Filter records with rate_limits, deduplicate consecutive identical (5h%, 7d%) pairs |
| `cctokens sl --per session` | Session summaries | Segment-aware aggregation per session_id |
| `cctokens sl --per project` | Project summaries | Aggregate session summaries by project |
| `cctokens sl --per day` | Daily summaries | Aggregate session summaries by day |
| `cctokens sl --per window` | 5h window summaries | Group by `five_hour_resets_at`, compute est budget |
| `cctokens sl --chart 5h` | 5h% braille chart | Y-axis: 0-100%, X-axis: time |
| `cctokens sl --chart 7d` | 7d% braille chart | Y-axis: 0-100%, X-axis: time |
| `cctokens sl --chart cost` | Cost braille chart | Y-axis: $, X-axis: time |

## Table Columns Per View

**Default (ratelimit):**
```
Time | 5h% | 7d% | 5h Resets | Session
```
Compact: hide Session.

**`--per session`:**
```
Session | Project | Cost | Duration | API Time | API% | Lines +/- | Ctx% | Segs
```
Compact: hide API Time, API%, Ctx%, Segs.

**`--per project`:**
```
Project | Cost | Duration | API Time | Sessions
```

**`--per day`:**
```
Date | Cost | Sessions | Peak 5h% | Peak 7d%
```

**`--per window`:**
```
5h Window | Peak 5h% | Sessions | Cost | Est Budget
```

**`--cost-diff` (adds to `--per session`):**
Replaces Cost column with:
```
Cost(SL) | Cost(LiteLLM) | Diff | Diff%
```

## CLI Flags

### New (sl-specific)

| Flag | Description |
|------|-------------|
| `--file <path>` | statusline.jsonl path. Default: `~/.claude/statusline.jsonl` |
| `--cost-diff` | Show cost comparison with cctokens LiteLLM calculation. Only with `--per session`. |

### Reused from main cctokens

| Flag | Reused as-is |
|------|-------------|
| `--from`, `--to` | Time filtering |
| `--5hfrom`, `--5hto`, `--1wfrom`, `--1wto` | Time window shortcuts |
| `--tz` | Timezone |
| `--session`, `--project`, `--model` | Substring filters |
| `--output` | json, csv, markdown, html, txt, tsv |
| `--filename` | Custom output file path |
| `--copy` | Clipboard |
| `--cost` | true / decimal / false |
| `--order` | asc / desc |
| `--table` | auto / full / compact |

### `--per` values for sl

`session`, `project`, `day`, `window` (different from main cctokens which uses `day`, `hour`, `month`, `session`, `project`, `model`).

### `--chart` values for sl

`5h`, `7d`, `cost` (different from main cctokens which uses `cost`, `token`).

## Error Handling

- **File not found**: clear error message with hint about statusline.sh setup
- **Empty file / no records after filter**: empty table + stderr message, exit 0
- **Malformed lines**: skip and count, report `Skipped N malformed lines` on stderr
- **No rate_limits in ratelimit view**: `No rate limit data found` message
- **`--cost-diff` without matching session**: show `—` for LiteLLM columns
- **`--cost-diff` without `--per session`**: stderr warning, ignore the flag
- **`--chart` with `--per`**: ignore `--per` (same as main cctokens)

## JSON Output Structure

```json
{
  "meta": {
    "source": "statusline",
    "file": "~/.claude/statusline.jsonl",
    "view": "ratelimit|sessions|project|day|window",
    "from": "...",
    "to": "...",
    "tz": "...",
    "generated_at": "..."
  },
  "data": [ ... ],
  "totals": { ... }
}
```

`data` contents match the current view mode. `totals` present for sessions/project/day/window modes. Absent for ratelimit mode.

## Cost Cross-Comparison

When `--cost-diff` is used with `--per session`:

1. Run the existing `load_records` + `calculate_cost` pipeline (using `--claude-dir`, `--from`, `--to`, `--tz` from the same invocation)
2. Group by session, sum total_cost per session
3. Match by session_id prefix (first 8 chars) against sl session summaries
4. Display both costs, absolute diff, and percentage diff

Observed difference: statusline self-reported cost is consistently ~2-4% higher than LiteLLM-calculated cost, likely due to minor pricing differences.
